# オプション・バックテスト (Option Backtesting)

オプションのバックテストは、株式や先物よりもデータ品質と約定仮定の影響が大きい。特にIV、板、流動性、SQ処理、権利行使価格の選定を雑に扱うと、成績が大きく歪む。

## 1. よくあるバイアス

| バイアス | 内容 | 対策 |
| :--- | :--- | :--- |
| **Stale quote** | 気配や理論価格が更新されていない | 出来高、更新時刻、スプレッドでフィルタ |
| **Mid約定仮定** | Bid/Ask中値で常に約定できる前提 | Bid/Ask、スリッページ、約定優先度を設定 |
| **Survivorship bias** | 現在残っている銘柄・限月だけで検証 | 過去の全限月・全strikeを含める |
| **Liquidity bias** | 板が薄いstrikeを大きく売買できる前提 | 出来高、建玉、売買代金、板数量で制限 |
| **Look-ahead bias** | 当時未公表のIV/清算値/財務情報を使う | データ公表時刻と利用可能時刻を保存 |
| **SQ処理ミス** | SQ決済、権利消滅、最終取引日を誤る | 限月ルールと最終清算値を明示的に処理 |

## 2. 価格データの優先順位

- **実約定価格**: 最も現実に近いが、約定が少ないstrikeでは欠損が多い。
- **Bid/Ask**: 約定可能性の評価に重要。中値だけでなくスプレッド幅を見る。
- **清算値**: 日次評価には便利だが、実際にその価格で約定できるとは限らない。
- **理論価格**: 欠損補完に使えるが、モデルとIV surfaceに依存する。

## 3. IV Surfaceの補間

- Strike方向はDeltaまたはmoneynessで正規化する。
- 限月方向は分散時間で補間する。
- 補間方法を途中で変えると、戦略成績が不連続になる。
- 外挿が必要な遠いOTM strikeは、流動性が低いため検証対象から外す判断も必要。

## 4. 約定・コスト仮定

- Bidで売り、Askで買う保守的な仮定を基本にする。
- スプレッドが広い場合は取引しないルールを入れる。
- 建玉や出来高に対する注文サイズ上限を設定する。
- 証拠金、手数料、税金、ロール時のコストを含める。

## 5. 検証指標

- CAGR、Sharpe Ratioだけでなく、最大ドローダウン、最大損失日、Expected Shortfallを見る。
- ショートオプション戦略では、勝率よりもテール損失と回復期間を重視する。
- IVレジーム別、SQ残存日数別、Skew水準別に成績を分解する。
